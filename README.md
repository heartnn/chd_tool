# 🚀 CHD 智能转换工具 (Smart CHD Tool)

### 专门解决中文路径乱码与镜像格式识别的终极方案

在玩老游戏镜像转换时，最痛苦的莫过于 `chdman` 不支持中文路径，或者分不清一个 `.iso` 到底该用 `createcd` 还是 `createdvd`。本工具通过 **Python 自动化脚本 + Windows API 注入 + 临时重命名技术**，彻底解决了这些痛点。

---

## ✨ 核心特性

* **🔍 智能内容识别**：
    * 不再单纯依赖文件后缀名。
    * 自动扫描镜像二进制头。如果 `.iso` 文件包含 `CD001` 标识且小于 900MB（如 PS1 游戏），将强制使用 `createcd` 模式，确保 100% 兼容。
    * 自动检测扇区大小（2352 字节整除检测），精准区分 CD 与 DVD。
* **🔤 完美支持中文路径**：
    * **Unicode 注入**：调用 Windows 底层 `GetCommandLineW` API，彻底解决拖放文件时的乱码问题。
    * **临时重命名技术**：在转换瞬间自动将中文文件名映射为临时英文字符，避开 `chdman` 对非 ASCII 字符的处理缺陷，处理完自动还原。
* **📂 CUE 链路保护**：
    * 不仅重命名 `.cue`，还会自动重命名其内部引用的 `.bin` 文件并动态修改 CUE 文本。解决“找不到 BIN 文件”的顽疾。
* **📦 单文件便携设计**：
    * 支持将 `chdman.exe` 打包进单文件 EXE，无需配置环境变量，双击即用。

---

## 🛠️ 如何使用？

### 方式 A：拖放操作 (推荐)
只需将一个或多个**文件**、甚至整个**文件夹**，直接拖拽到 `chd_tool.exe` 上：
1.  如果是压缩镜像（`.iso`, `.cue`, `.bin`, `.gdi`, `.img`），它会自动转为 `.chd`。
2.  如果是解压 `.chd`，它会智能识别内部格式并还原为 `.cue` 或 `.iso`。

### 方式 B：批量扫描
将 `chd_tool.exe` 放在你的游戏库根目录下，直接双击运行。它会自动递归扫描所有子目录，把没转换的镜像全部处理掉。

## 🔨 打包教程 (针对开发者)

如果你想自己修改源码并打包成单文件：

1.  **安装环境**：
    `pip install pyinstaller`
2.  **准备文件**：
    确保 `chd_tool.py` 和 `chdman.exe` 在同一目录下。
3.  **运行打包命令**：
    ```bash
    pyinstaller -F -c --add-data "chdman.exe;." chd_tool.py
    ```
    *打包成功后，在 `dist` 文件夹下即可找到最终的单文件。*

## 📝 脚本逻辑说明

针对 `.cue` 文件的处理流程如下：
1.  **读取编码**：尝试以 UTF-8, GBK 等多种编码读取 `.cue` 文本。
2.  **映射 BIN**：提取 `FILE "xxx.bin" BINARY` 中的文件名。
3.  **双重重命名**：将 `.cue` 改名为 `t_01.cue`，`.bin` 改名为 `t_01.bin`。
4.  **动态修正**：修改 `t_01.cue` 里的内容使其指向 `t_01.bin`。
5.  **核心调用**：命令 `chdman` 处理 `t_01.cue`。
6.  **还原现场**：无论成功失败，所有文件名和文件内容全部归位。

---

## ⚠️ 注意事项
* 请确保 `.cue` 文件中记录的文件名与实际的 `.bin` 文件名一致（包括繁简区别）。
* 本工具基于 `chdman 0.287` 优化，建议配合最新版内核使用。
